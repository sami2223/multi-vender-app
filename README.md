## Mini Multi-Vendor Product Management

Requirements implemented:
- Roles: `vendor`, `admin`
- Vendors manage their own products; admin approves/rejects
- Service layer: `App\\Services\\ProductService`
- Helper: `generateProductCode()`
- Event: `ProductCreated` and queued notification to admins
- Queue: database driver with jobs table migration present
- Seeders: 1 admin, 1 vendor, 2 products

### Setup

1) Configure `.env` for DB and mail (copy from example if needed)
```
cp .env.example .env
```

2) Install dependencies and generate key
```
composer install
php artisan key:generate
```

3) Migrate and seed
```
php artisan migrate --seed
```

4) Use database queue
```
QUEUE_CONNECTION=database
```

5) Run server and worker
```
php artisan serve
php artisan queue:work
```

### Default accounts
- Admin: admin@example.com / password
- Vendor: vendor@example.com / password

### Routes
- Vendor:
  - GET `/products`
  - POST `/products`
  - PUT `/products/{product}`
  - DELETE `/products/{product}`
- Admin:
  - GET `/admin/products`
  - POST `/admin/products/{product}/approve`
  - POST `/admin/products/{product}/reject`
